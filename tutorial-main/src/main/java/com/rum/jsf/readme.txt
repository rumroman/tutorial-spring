                                            Java Server Faces.

По сути это тоже самое , что и JSP но в более удобном виде. В JSF отсутствует смешивание кода в java с HTML.
У JSF есть собственный компонент который отвечает за жизненный цикл , за сохранность компонентов.

JSF - это компонентно-ориентированная разработка.
JSF - это swing для веб-программирования.

JSF - множество собственных компонентов , но существует возможность создания своих компонентов.

JSF - это спецификация , у которой существует как собственная ("Mojarra"), так и сторонние реализации
      (множество библиотек с дополненными компонентами, RichFaces,MyFaces и др.)


Разделения представления(веб страницы) и бизнес-логики(JavaBeans).

Использование паттерна MVC - Model-Viewer-Controller.

Использование синтаксиса Facelets:

-синтаксис страницы только в формате XHTML,
-использование тегов JSF (<h:form>,<h:head> и др.),
-создание шаблонов страниц.
