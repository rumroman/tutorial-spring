                                        JSP.

JSP страница такая же как HTML за исключением того, что в ней еще содержится Java код.

JSP - это сервлет.  Компилируется в сервлет. Задача JSP - с помощью данных, который написал программист
нарисовать HTML-страничку.

request = Объект HttpServletRequest, связанный с запросом, котоырй позволяет вам обращаться к параметрам
запроса ( через метод getParameter), типу запроса (GET, POST, HEAD и т.д) , и входящим HTTP заголовком
(cookies, Refer, и тд).

response = Объект HttpServletResponse, связанный с ответом на зпрос клиента. Обратите внимание что,
поскольку поток вывода буферизован, можно изменять коды состояния HTTP и заголовки ответов, даже если
это недопустимо в обычном сервлете , но лишь в том случае если какие-то данные вывода уже были отправлены
клиенту.

session = обхект типа HttpSession , связаный с запросом. Сессии создаются автоматически , и эта переменная
существует даже если нет ссылок на входящие сессии. Единственным исключением является ситуация , когда вы
отключаете использование сессий используя атрибут session директивы page. В этом случае ссылки на переменную
session приводят к возникновению ошибок при трансляции JSP страницы в сервлет.


JSTL - JSP Standard Tag Library - библиотека стандартных тэгов JSP.
